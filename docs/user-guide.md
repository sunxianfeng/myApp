## 快速开始

### 登录

1. 打开应用：http://localhost:5173
2. 输入用户名和密码登录
3. 进入主页面

### 仪表盘

仪表盘展示：
- 最近生成的试卷
- 题目统计信息
- 常用操作快捷方式

## 题目管理

### 查看题目

1. 点击左侧菜单 "题目管理"
2. 查看题目列表
3. 使用搜索和筛选功能查找特定题目

### 创建题目

1. 点击 "新增题目" 按钮
2. 填写题目信息：
   - **题目内容**：题目的具体文字
   - **题目类型**：单选、多选、填空、简答等
   - **分类**：按学科或章节分类
   - **选项**：（针对选择题）输入各选项
   - **答案**：标记正确答案
   - **难度**：简单、中等、困难
   - **出题人**：题目的出题者信息

3. 点击 "保存" 完成创建

### 编辑题目

1. 在题目列表中找到要编辑的题目
2. 点击 "编辑" 按钮
3. 修改相关信息
4. 点击 "保存" 完成修改

### 删除题目

1. 选择要删除的题目
2. 点击 "删除" 按钮
3. 确认删除

### 批量操作

1. 选择多个题目（勾选复选框）
2. 使用批量操作菜单：
   - 批量导出
   - 批量修改分类
   - 批量删除

### 导入题目

#### 方式一：文件上传

1. 点击 "导入题目"
2. 选择文档文件（Word、PDF或TXT）
3. 系统自动解析题目
4. 查看解析结果并确认导入

#### 方式二：手动输入

1. 使用 "新增题目" 逐个输入题目

## 模板设置

### 查看模板

1. 点击左侧菜单 "模板管理"
2. 查看可用的模板列表
3. 点击模板可查看详细信息

### 创建模板

1. 点击 "新增模板" 按钮
2. 填写模板信息：
   - **模板名称**：如"高考数学模拟试卷"
   - **模板描述**：模板的简要说明
   - **难度配置**：
     - 简单题比例
     - 中等题比例
     - 困难题比例
   - **题型配置**：
     - 各题型的题目数量
     - 各题型的分值

3. 点击 "保存" 完成创建

### 编辑模板

1. 在模板列表中找到要编辑的模板
2. 点击 "编辑" 按钮
3. 修改模板配置
4. 点击 "保存" 完成修改

### 删除模板

1. 选择要删除的模板
2. 点击 "删除" 按钮
3. 确认删除

## 试卷生成

### 生成试卷

1. 点击左侧菜单 "试卷生成"
2. 选择一个模板（或保持默认）
3. 点击 "开始生成" 按钮
4. 系统根据模板自动生成试卷

### 查看试卷

1. 点击左侧菜单 "我的试卷"
2. 查看已生成的试卷列表
3. 点击试卷可查看详细内容

### 编辑试卷

1. 打开试卷
2. 点击 "编辑" 按钮
3. 可以：
   - 调整题目顺序（拖拽）
   - 移除不需要的题目
   - 添加新题目
   - 修改试卷标题和说明

4. 点击 "保存" 完成编辑

### 导出试卷

1. 打开试卷
2. 点击 "导出" 按钮
3. 选择导出格式：
   - **Word** - 用于打印
   - **PDF** - 用于在线查看
   - **Excel** - 用于数据分析

4. 下载导出的文件

### 打印试卷

1. 打开试卷
2. 使用浏览器打印功能（Ctrl+P 或 Cmd+P）
3. 调整打印设置
4. 点击 "打印"

## 常见问题

### Q: 如何导入大量题目？

A: 您可以：
1. 准备一个Word或PDF文档，包含所有题目
2. 使用 "导入题目" 功能上传文档
3. 系统自动解析并导入题目

### Q: 如何修改已生成的试卷？

A: 
1. 打开要修改的试卷
2. 点击 "编辑" 按钮
3. 进行修改（调整题目顺序、添加/删除题目等）
4. 点击 "保存"

### Q: 试卷生成失败怎么办？

A:
1. 检查选择的模板是否正确
2. 确保题库中有足够的题目
3. 查看浏览器控制台的错误信息
4. 联系管理员或技术支持

### Q: 如何备份我的数据？

A:
1. 联系系统管理员
2. 管理员可以使用备份脚本：`./scripts/backup.sh`
3. 备份文件保存在 `data/backups/` 目录

### Q: 支持哪些文件格式导入？

A: 目前支持以下格式：
- Word (.docx, .doc)
- PDF (.pdf)
- 纯文本 (.txt)

### Q: 如何获得技术支持？

A:
1. 查看 [API文档](api.md) 了解技术细节
2. 查看 [部署指南](deployment.md) 了解部署问题
3. 提交Issue到项目仓库
4. 联系开发团队
